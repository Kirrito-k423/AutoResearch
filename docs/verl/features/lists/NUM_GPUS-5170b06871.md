# NUM_GPUS

- **参数名**：`NUM_GPUS`
- **分类**：效率
- **中文解释**：机制推断：单节点启动的 GPU 进程数，脚本传给 `torchrun --nproc_per_node`，决定每个节点参与训练的本地 rank 数。
- **常见值**：8
- **来源环境变量**：NUM_GPUS
- **性能影响**：机制推断：增加 GPU 数通常提高并行吞吐和可容纳模型/批量规模，但也增加通信和同步开销；需要与 TP/PP/EP/CP 等并行配置匹配。
- **精度影响**：机制推断：若全局 batch、学习率和随机种子保持等价，GPU 数本身不改变目标；实际并行规模变化常会改变 batch 组成和规约顺序，从而影响训练轨迹。
- **NPU/Ascend 证据**：未知
- **CI 看护**：部分
- **示例数**：2
- **需要子代理补证**：否

## 示例脚本

- `examples/sft/gsm8k/run_mimo_7b_mtp_megatron.sh`
- `examples/sft/gsm8k/run_qwen3_5_397b_a17b_megatron.sh`

## 证据片段

- `examples/sft/gsm8k/run_mimo_7b_mtp_megatron.sh:4` NUM_GPUS=${NUM_GPUS:-8}
- `examples/sft/gsm8k/run_qwen3_5_397b_a17b_megatron.sh:37` NUM_GPUS=${NUM_GPUS:-8}
