# VAL_N

- **参数名**：`VAL_N`
- **分类**：效率
- **中文解释**：控制验证阶段每个 prompt 生成的候选数，映射到 `actor_rollout_ref.rollout.val_kwargs.n`；示例默认 1。
- **常见值**：1
- **来源环境变量**：VAL_N
- **性能影响**：机制推断：验证生成成本、KV cache 占用和日志体积通常随 `n` 近似线性增长；较大的 `n` 会延长 test/validation 阶段并占用 rollout 资源。
- **精度影响**：文档说明：Verl best practices 建议小测试集可增大验证 `n` 以降低指标方差；但这会改变评测口径，需与历史结果保持一致。
- **NPU/Ascend 证据**：未知
- **CI 看护**：部分
- **示例数**：1
- **需要子代理补证**：否

## 示例脚本

- `examples/grpo_trainer/run_qwen3_30b_a3b_megatron.sh`

## 证据片段
