# REF_EP

- **参数名**：`REF_EP`
- **分类**：效率
- **中文解释**：文档说明：reference policy 的 Megatron expert parallel size，示例写入 `actor_rollout_ref.ref.megatron.expert_model_parallel_size`，通常默认跟随 actor 的 MoE EP。
- **常见值**：${actor_ep
- **来源环境变量**：REF_EP
- **性能影响**：机制推断：增大 EP 可分摊 MoE expert 权重和计算，降低单卡显存压力；代价是专家路由、all-to-all 通信和调度复杂度上升。
- **精度影响**：机制推断：正确切分应保持 reference policy 语义；EP 与模型专家数/检查点不匹配时可能导致加载失败或数值不一致。
- **NPU/Ascend 证据**：未知
- **CI 看护**：部分
- **示例数**：1
- **需要子代理补证**：否

## 示例脚本

- `examples/grpo_trainer/run_qwen3_30b_a3b_megatron.sh`

## 证据片段

- `examples/grpo_trainer/run_qwen3_30b_a3b_megatron.sh:48` ref_ep=${REF_EP:-${actor_ep}}
