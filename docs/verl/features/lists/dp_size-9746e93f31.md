# dp_size

- **参数名**：`dp_size`
- **分类**：效率
- **中文解释**：文档说明：示例中的数据并行规模变量；在 VeOmni 脚本里写入 actor `fsdp_size`，NPU 参数表也把 `dp_size` 对应 rollout `data_parallel_size`。
- **常见值**：16
- **来源环境变量**：dp_size
- **性能影响**：机制推断：增大 DP/FSDP 规模可分摊数据和参数分片、提升总吞吐，但需要更多设备并增加跨卡同步；作为 `fsdp_size` 使用时还会改变分片组大小和通信范围。
- **精度影响**：机制推断：正确数据并行不改变优化目标；但全局 batch、梯度同步、FSDP 分片组或 checkpoint mesh 不匹配时，可能影响训练稳定性或导致恢复失败。
- **NPU/Ascend 证据**：未知
- **CI 看护**：部分
- **示例数**：1
- **需要子代理补证**：否

## 示例脚本

- `examples/grpo_trainer/run_qwen3_vl_30b_moe_veomni.sh`
