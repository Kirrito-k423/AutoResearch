# REF_PP

- **参数名**：`REF_PP`
- **分类**：效率
- **中文解释**：文档说明：reference policy 的 Megatron pipeline parallel size，示例写入 `actor_rollout_ref.ref.megatron.pipeline_model_parallel_size`，通常默认跟随 actor 的 PP。
- **常见值**：${actor_pp
- **来源环境变量**：REF_PP
- **性能影响**：机制推断：增大 PP 可把 reference 模型层切到更多设备上，降低单卡显存；代价是流水线气泡、跨 stage 通信和调度复杂度增加。
- **精度影响**：机制推断：正确 pipeline 切分应保持 logprob 等价；切分与检查点或层数不匹配会导致加载/运行失败或数值差异。
- **NPU/Ascend 证据**：未知
- **CI 看护**：部分
- **示例数**：1
- **需要子代理补证**：否

## 示例脚本

- `examples/grpo_trainer/run_qwen3_30b_a3b_megatron.sh`

## 证据片段
