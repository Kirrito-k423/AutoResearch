# NODE_RANK

- **参数名**：`NODE_RANK`
- **分类**：效率
- **中文解释**：机制推断：`torchrun` 多节点启动中的当前节点编号，示例传给 `--node_rank`，用于让每个节点在同一个分布式作业中定位自己的 rank。
- **常见值**：0
- **来源环境变量**：NODE_RANK
- **性能影响**：机制推断：正确设置后不直接影响吞吐；设置重复或错误会导致进程组初始化失败、通信挂起或节点角色错乱。
- **精度影响**：机制推断：不直接影响训练精度；错误的 rank 配置通常表现为启动失败或通信异常，而不是可控的精度调节。
- **NPU/Ascend 证据**：未知
- **CI 看护**：部分
- **示例数**：1
- **需要子代理补证**：否
