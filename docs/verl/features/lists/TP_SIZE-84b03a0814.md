# TP_SIZE

- **参数名**：`TP_SIZE`
- **分类**：效率
- **中文解释**：文档说明：`TP_SIZE` 是 examples 中的 tensor model parallel size，写入 SFT/Megatron engine 的 `tensor_model_parallel_size`，用于把线性层等张量维度切到多个 rank。
- **常见值**：1、2、8
- **来源环境变量**：TP_SIZE
- **性能影响**：文档说明：Verl 性能建议中 TP 是显存与通信的核心权衡；增大 TP 可降低单卡参数/激活压力、支持更大模型，但增加层内 all-reduce/all-gather 通信。
- **精度影响**：机制推断：不直接改变训练目标；不同切分和通信顺序可能带来微小浮点差异，主要影响是能否使用目标 batch/模型规模。
- **NPU/Ascend 证据**：未知
- **CI 看护**：部分
- **示例数**：4
- **需要子代理补证**：否

## 示例脚本

- `examples/sft/gsm8k/run_mimo_7b_mtp_megatron.sh`
- `examples/sft/gsm8k/run_nemotron_nano_v3_megatron.sh`
- `examples/sft/gsm8k/run_qwen3_5_397b_a17b_megatron.sh`
- `examples/sft/vlm/run_qwen3_vl_2b_fsdp.sh`
