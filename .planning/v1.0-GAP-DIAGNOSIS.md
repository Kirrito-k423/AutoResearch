## Fixes Applied

1. `workspace-core/ssh/tunnel.py`
   - Added `ControlMaster=no` and `ControlPath=none` to the `ssh -N -R`
     command.
   - Reason: user SSH config may enable ControlMaster. In that case `ssh -R`
     can return `rc=0` immediately while a mux master owns the actual remote
     forward, leaving AutoResearch without a manageable tunnel pid.

2. `autoresearch/net/tunnel.py`
   - Added one-shot stale remote forward cleanup when `ssh -R` reports
     `remote port forwarding failed`.
   - The cleanup connects through the configured server, looks only for `sshd`
     listeners on the requested tunnel port, terminates those stale listeners,
     and then lets the existing bounded retry loop rebuild the tunnel.

3. `autoresearch/hw/probe.py`
   - Changed hardware SSH connect to `connect_timeout=5.0, retries=0`.
   - Reason: the acceptance requirement is a 5 second connection timeout with
     readable failure output; default SSH retries can make a single unreachable
     host take much longer than that.

4. Tests
   - Added tunnel command assertions for `ControlMaster=no` and
     `ControlPath=none`.
   - Added stale remote-forward cleanup coverage.
   - Added hardware connect retry assertion.
