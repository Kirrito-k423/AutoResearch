# Phase 10 Plan 03: 本机 Archon 验证 + Web UI 可观察性

Plan 03 负责把 Phase 10 从“文件写好了”推到“本机真的能跑、UI 真的能看”。这一步也顺手把 README / 服务文档更新成用户可直接照着走的版本。

## Tasks

1. **本机 Archon CLI / 服务准备**
   - 检查 `archon` CLI 是否存在；若不存在，按官方安装路径补装
   - 启动或恢复 `archon serve`
   - 用 `curl http://localhost:8088/healthz` 验证本机 Archon health

2. **Workflow 静态验证**
   - 运行：
     - `archon validate workflows`
     - 必要时 `archon workflow list`
   - 修复 validator 指出的 schema / runtime / path 问题
   - 确认 repo-local `.archon/workflows/` 都被识别到

3. **主 workflow 真实运行**
   - 用 live checkout 跑主 workflow，优先：
     - `archon workflow run ar-min-loop --no-worktree ""`
   - 如需显式输入，使用环境变量覆盖：
     - `AR_CONFIG_PATH=config/config.yaml`
     - `AR_SERVER=<real UAT server>`
     - `AR_LIB=verl`
     - `AR_PUSHGATEWAY_URL=http://127.0.0.1:17891`
   - 至少记录：
     - workflow run 是否成功
     - collect 产出的 `run_id`
     - report 路径
     - 失败时卡在哪个节点、什么错误

4. **Web UI 可观察性验证**
   - 在 Archon Web UI 中确认：
     - workflow run 出现在运行列表
     - 节点执行顺序与 YAML 一致
     - progress / result cards 可见
   - 若本机已装 Browser/Chrome 工具或可用截图链路，留下 UI 侧证据；否则至少在 UAT 文档中写清观察结果

5. **文档与 planning 状态更新**
   - `services/archon/README.md`
     - 补 `archon validate workflows`
     - 补 `ar-min-loop` 的运行例子
   - `services/README.md` / `README.md`
     - 同步 quickstart 和 Phase 10 后的推荐验证顺序
   - `.planning/phases/10-archon/10-UAT.md`
     - 写真实命令、真实结果、run id、UI 观察结论
   - `.planning/phases/10-archon/10-VERIFICATION.md`
     - 对 `ARCH-WF-*` / `ARCH-RUN-*` 做逐条核对
   - `.planning/REQUIREMENTS.md` / `.planning/ROADMAP.md` / `.planning/STATE.md`
     - Phase 10 完成后同步打勾与状态推进

## 风险

- **R1 Archon 安装源或本机 PATH 问题**：需要预留一次安装/重启的时间，不把“命令不存在”误判成 workflow 自身故障。
- **R2 `workflow run` 需要 prompt 参数**：若 CLI 仍要求 prompt，使用空字符串 `""` 或 README 中约定的最小 prompt，不在 phase 里另造输入协议。
- **R3 Web UI 证据不完整**：如果当前自动化工具无法直接截图 UI，至少要把观察到的 run id、节点顺序、页面状态写进 UAT/VERIFICATION，避免只留一句“看起来可以”。
