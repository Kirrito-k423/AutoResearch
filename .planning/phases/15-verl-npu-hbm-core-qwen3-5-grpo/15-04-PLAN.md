# Phase 15 Plan 04: Numbered Data Repository Bundle And Rebuildable Visualizations

## Objective

Reorganize formal Verl delivery artifacts into an `autoresearch-log` data repository layout that is readable, portable, and sufficient to rebuild W&B and Prometheus-style visualizations from copied data.

## Threat Model

- **T-15-04-01 mixed code/data state:** Experiment logs must not be hidden under the AutoResearch code repository or `~/.autoresearch`.
- **T-15-04-02 non-rebuildable dashboard:** W&B and Prometheus views must be reconstructable from saved artifacts, not only from live service state.
- **T-15-04-03 unreadable delivery:** The top-level bundle must guide a customer from report to raw evidence in priority order.
- **T-15-04-04 provenance drift:** Immutable config and code commit ids must remain beside the data that came from that code.
