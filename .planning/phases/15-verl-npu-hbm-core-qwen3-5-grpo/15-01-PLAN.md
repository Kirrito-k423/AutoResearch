# Phase 15 Plan 01: Runtime NPU HBM/Core Telemetry And Prometheus Evidence

## Objective

Add continuous per-case NPU telemetry for formal Verl runs so HBM, AI Core, and NPU utilization curves can be inspected and rebuilt from saved artifacts.

## Threat Model

- **T-15-01-01 false resource evidence:** A single end-of-run probe can miss peak HBM or core utilization. Sampling must cover the whole case lifetime.
- **T-15-01-02 sampler masks training failure:** Sampler startup or parsing problems must not hide the Verl process exit code.
- **T-15-01-03 unjoinable metrics:** Telemetry without run/case/server/device labels cannot be correlated with matrix rows or W&B runs.
- **T-15-01-04 overclaiming 0.5s:** Native `npu-smi info watch` on A2-AK-225 supports only integer seconds; report 1s native sampling unless a later safe API proves otherwise.
