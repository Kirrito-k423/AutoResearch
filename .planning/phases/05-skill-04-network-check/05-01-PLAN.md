<threat_model>
  <threat id="T-05-01-01" severity="HIGH">
    <asset>本机 shell 与远程 shell</asset>
    <attack>恶意 target URL 通过 config 进入 curl shell 命令，造成命令注入或读取非 HTTP 资源</attack>
    <mitigation>target URL 只允许 http/https；本地 curl 使用 subprocess argv；远程命令使用安全 quoting；拒绝其他 scheme</mitigation>
    <verification>`tests/test_net_curl.py` 覆盖 `file://`、`;`、换行和空 URL 拒绝；`tests/test_net_probe.py` 断言远程命令包含安全 quoting</verification>
  </threat>
  <threat id="T-05-01-02" severity="MEDIUM">
    <asset>诊断结果可信度</asset>
    <attack>curl stderr、重定向或失败输出污染解析，导致失败目标被误判成功</attack>
    <mitigation>`curl -w` 使用 delimiter-safe 输出；解析 http_code/time_total/speed_download；非零 exit code 和无法解析均形成失败 attempt</mitigation>
    <verification>curl parser 单测覆盖 200、302 后成功、timeout、DNS 失败和 malformed write-out</verification>
  </threat>
  <threat id="T-05-01-03" severity="HIGH">
    <asset>配置秘密与 stdout 协议</asset>
    <attack>配置中的 identity_file/bootstrap secret 或进度日志进入最终 stdout JSON</attack>
    <mitigation>rows 只写 target/server name 和网络指标；stdout 只在 run_probe 末尾写一次 JSON；progress 走 stderr</mitigation>
    <verification>`tests/test_net_cli.py` 对 stdout 做单次 `json.loads`，并断言敏感键和 `__AR_PROGRESS__=` 不在 stdout</verification>
  </threat>
</threat_model>

创建 `autoresearch.net` 包。定义 `CurlAttempt`、`NetworkRow`、`NetworkSummary` 类型，字段至少包含 `location/server/target_label/target_url/effective_mode/status/http_code/latency_ms/speed_download_bps/attempts/error`。

实现 `validate_target_url(url: str) -> str`，仅允许 `http` 和 `https` scheme，空 host、非 HTTP scheme、含换行或控制字符的 URL 视为配置错误。

实现 `build_curl_command(url, proxy_url=None, timeout_s=10) -> list[str]`，返回 subprocess argv：`curl --max-time 10 -L -o /dev/null -s -w <format> <url>`。format 必须能稳定解析 `http_code`、`time_total` 和 `speed_download`，且不依赖 HTML body。

实现 `parse_curl_result(exit_code, stdout, stderr, mode, target_url) -> CurlAttempt`。`latency_ms` 从 `time_total` 转整数毫秒；`speed_download_bps` 从 curl 的 bytes/sec 转整数；非零 exit code 或缺失字段必须保留 stderr 摘要并标失败。
  </action>
  <verify>
    <automated>uv run pytest -q tests/test_net_curl.py</automated>
  </verify>
  <acceptance_criteria>
    - `validate_target_url("https://github.com")` 通过，`file:///etc/passwd` 和含换行 URL 抛配置/值错误
    - `build_curl_command("https://baidu.com")` 返回 list argv，包含 `--max-time`、`10`、`-L`、`-o`、`/dev/null`、`-s`、`-w`
    - 200 成功 write-out 解析出 `http_code=200`、整数 `latency_ms`、整数 `speed_download_bps`
    - timeout/DNS failure 产生失败 attempt，stderr 摘要有界，不抛未处理异常
    - `autoresearch/net/curl.py` 不 import SSHClient、Click 或 config loader
  </acceptance_criteria>
