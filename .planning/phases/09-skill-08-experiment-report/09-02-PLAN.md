# Phase 09 Plan 02: HTML 模板 + 嵌入图表 + 浏览器打开

Plan 02 负责把 `ReportBundle` 落成一个真正可读、可打开、可跳转的本地 `report.html`，并挂到顶层 CLI 上。

## Tasks

1. **`autoresearch/report/render.py`**
   - `render_report(bundle: ReportBundle, output_path: Path) -> Path`
   - 产出单文件静态 HTML
   - 页面结构至少包含：
     - 顶部 summary cards（run_id / server / lib / elapsed / exit_code / artifact completeness）
     - `log` 区块（关键行摘要 + 末尾片段 + `log.txt` 链接）
     - `wandb` 区块（summary metrics + 单点图/迷你图 + wandb 链接）
     - `prometheus` 区块（current metric + 单点图/迷你图 + query 链接）
     - `raw artifacts` 区块（manifest / wandb dir / log）
   - 不依赖新本地服务；CSS/JS 全内联或极小依赖

2. **图表策略**
   - 用最轻量方案渲染 snapshot 图：
     - 可选简单 inline SVG / 微型 canvas / 极小原生 JS
   - 重点不是“图多复杂”，而是让三路视图一眼可扫
   - 若无数据则显示 unavailable 状态，而不是空白容器

3. **`autoresearch/report/cli.py`**
   - `autoresearch report render --run-id X [--open]`
   - CLI 最终 stdout 仍保持唯一 JSON，至少含：
     - `ok`
     - `run_id`
     - `report`
     - `opened`
     - `warnings`
   - `--open` 走系统默认浏览器，失败只记 warning

4. **顶层 CLI 接线**
   - 在 `autoresearch/cli.py` 注册 `report` group
   - 帮助文案保持中文，风格与现有 `collect` / `services` group 对齐

5. **测试**
   - `tests/test_report_render.py`
     - bundle -> report.html
     - HTML 至少包含 run summary / log / wandb / prom / links
   - `tests/test_report_cli.py`
     - happy path 输出 JSON
     - `--open` 调用被 mock 掉也能返回 `opened=true/false`
     - partial bundle 仍然生成 HTML

6. **UAT**
   - 用真实 run `01KV5MV7N5A3RBZ6388E5HCYAP` 跑：
     - `uv run autoresearch report render --run-id 01KV5MV7N5A3RBZ6388E5HCYAP --open`
   - 验 `~/.autoresearch/runs/<id>/report.html` 存在
   - 浏览器里能看到 run 摘要、log 摘要、wandb/promptheous 面板和跳转链接

## 风险

- **R1 页面内容过空**：Phase 8 只有 snapshot 数据，所以必须把摘要卡与 artifact links 设计成一等公民，不能等着“长曲线”撑页面。
- **R2 `--open` 跨平台差异**：优先用 Python 标准能力 / Click 现成能力，避免手写平台分支太多。
- **R3 HTML 模板过重**：本 phase 目标是本地复盘页，不是前端框架；保持单文件、轻依赖、可离线打开。
