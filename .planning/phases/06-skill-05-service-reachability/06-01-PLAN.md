# Phase 06 Plan 01: pushgateway 容器 + prometheus scrape job

引入 `prom/pushgateway` 容器, 让远程服务器 (Phase 6/8) 能把 metric push 到本机 9091, Prometheus 自动 scrape.
本 plan 不实现 reach test 探测, 只做"基础设施就绪".

## Tasks

1. **追加 pushgateway 容器** (`services/prometheus/compose.yml`)
   - image: `prom/pushgateway:v1.10.0`
   - container_name: `ar-pushgateway`
   - ports: `"${PORT_PUSHGATEWAY:-9091}:9091"`
   - 命令: `--persistence.file=/data/pushgateway.data --persistence.interval=1m` (重启 metric 不丢)
   - volumes: `pushgateway-data:/data`
2. **prometheus.yml 追加 scrape job**
   ```yaml
   - job_name: pushgateway
     honor_labels: true  # 保留 push 时打的 server/job 标签
     static_configs:
       - targets: ["pushgateway:9091"]
         labels:
           instance: pushgateway-local
   ```
3. **`autoresearch services start/stop/status` 集成**
   - start: 串行 `docker compose -f services/prometheus/compose.yml up -d` (新加的 pushgateway 自动起来)
   - status: 增加端口 9091 健康检查 (`curl -sf http://127.0.0.1:9091/-/healthy`)
   - stop: 现有 `docker compose down` 包含 pushgateway (无需改)
4. **测试**
   - `tests/test_services.py`: mock `docker compose` subprocess, 验证 pushgateway 容器在期望列表
   - 集成测试 (e2e): 实际 `docker compose up -d` 起服务, 验证 `curl -sf http://127.0.0.1:9091/-/healthy` 200
5. **UAT 文档** (`06-01-SUMMARY.md`):
   - 真机本地起服务: `autoresearch services start`, 确认 5 容器 (wandb/prom/grafana/pushgateway/...) 全 healthy
   - `curl -X PUT http://127.0.0.1:9091/metrics/job/test/instance/uat --data-binary 'foo 1'` 写入 metric
   - `curl -sf 'http://127.0.0.1:9090/api/v1/query?query=foo'` Prometheus 查得到 (间隔 ≤ 15s)

## 风险

- **D-36.B4 pushgateway persistence**: 用 `/data` volume 防止容器重启丢 metric; 若主机 bind mount 失败 fallback tmpfs
- **D-36 端口冲突**: 9091 选默认; `services/.env.example` 不预设, 默认值够用
