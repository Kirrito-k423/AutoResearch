# Phase 07 Plan 02: 1-step 干跑 (D-41, **NPU 适配**)

落地 `ar-stack check --server X` 的 1-step 干跑部分. 远程跑 NPU 算子 + verl/veomni import 验通.
**关键: 1-step 脚本走 torch_npu (NPU 适配), 不走 verl.trainer.main (CUDA 风格, 不适用).**

## Tasks

1. **`run_one_step_dryrun(spec, lib_name, conda_env)` 实现**
   - 构造远程命令:
     ```bash
     conda run -n <env> python -c "import torch, torch_npu, <lib>; \
       x = torch.randn(2,3).npu(); \
       y = (x+1).sum(); \
       print('SUM=', y.item()); \
       print('NPU_COUNT=', torch_npu.npu.device_count())"
     ```
   - 调 `client.exec(command, timeout=30.0)` (1-step 单独 30s, 不沿用 12s)
   - 解析 stdout: `SUM=<float>` + `NPU_COUNT=<int>`
2. **D-42 失败诊断**
   - 1-step 30s timeout -> CheckResult fail, error 含 "NPU 通信问题, 先跑 autoresearch hw probe --server X"
   - 1-step exit != 0 -> error 含 "torch_npu / verl/veomni import 失败, 检查 env 内 pip install"
   - 1-step stdout 无 `SUM=` -> error 含 "1-step 干跑脚本输出格式错, 报 bug"
3. **StackCheck 模型加 `npu_device_count` 字段** (`autoresearch/stack/models.py`)
4. **测试** (`tests/test_stack_checker.py`)
   - mock SSHClient, fixture 覆盖:
     - 1-step 成功: stdout 含 `SUM=6.0` + `NPU_COUNT=8` -> ok=True, npu_device_count=8
     - 1-step timeout (30s): mock client.exec raise CommandTimeoutError -> fail, error 含 "NPU 通信"
     - 1-step exit != 0 (e.g. ImportError) -> fail, error 含 "import 失败"
     - 1-step stdout 无 `SUM=`: fail, error 含 "输出格式错"
5. **UAT** (`07-02-SUMMARY.md`):
   - 4 台真机 `ar-stack check --server X`, 记录 1-step 结果 (NPU count, 干跑 latency)
   - 至少 1 台验证 SUM + NPU_COUNT 实际打印 (凭日志)

## 风险

- **D-41 NPU 适配**: torch_npu 在某些 env 装而不用 (verl 没声明), import 仍 OK 但 1-step 报 "no npu device" — 这种 stack 标 WARN 而非 FAIL (留给用户判断)
- **D-41.C4 30s timeout**: 远程冷启 torch_npu 可能 30s 不够, 后续如超 30s, 改 60s + warning
- **conda run**: 在某些机器 conda run 不可用, fallback 走 `source activate <env> && python -c "..."`
