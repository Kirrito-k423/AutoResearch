# Phase 07 Plan 01: conda_env 字段 + 库检测 (D-39, D-40, D-42 基础)

config 加 `conda_env` 字段; 落地 `autoresearch stack` CLI group + 库检测 (D-39) + conda env 探测 (D-40).
本 plan 不做 1-step 干跑 (D-41 是 07-02).

## Tasks

1. **`ServerSpec.conda_env` 字段** (`workspace-core/config/schema.py`)
   - 加 `conda_env: str = Field(default="", description="...")`, 默认空字符串 (不强制配)
   - 导出 `from workspace_core.config import ServerSpec` 已有
2. **`autoresearch/stack/` 模块骨架**
   - `__init__.py` (空) + `models.py` (StackCheck TypedDict) + `checker.py` (核心)
3. **`checker.py` 核心**
   - `check_library(spec, lib_name: str) -> StackCheck` 走 `python -c "import <lib>; print(<lib>.__version__)"`, 走 `conda run -n <env>` (有 env 时)
   - `probe_conda_env(spec, env_name: str) -> StackCheck` 走 `conda env list` 解析
   - `check_stack(spec) -> StackResult` 合并 library check + env probe
4. **CLI 接入** (`autoresearch/cli.py`)
   - `@main.group(name="stack")` 新 CLI group
   - `@stack.command("check")` 接 `--server`, `--config`, `--lang`
5. **config.example.yaml 更新**
   - 4 台 server 配 `conda_env: verl-env` / `veomni-env` (注释: 用户实际配时按 conda env 实际名)
6. **测试**:
   - `tests/test_stack_checker.py`: mock SSHClient 跑通 import OK / ImportError / conda env 不存在 3 路径
   - `tests/test_stack_cli.py`: CliRunner 测 CLI happy + help
   - `tests/test_config_validate.py`: 验证 conda_env 字段在 Pydantic 中可空可设
7. **UAT** (`07-01-SUMMARY.md`):
   - 4 台真机跑 `ar-stack check --server X`, 验 conda env 探测 + import 检测 (实际 1-step 不在这 plan)

## 风险

- **D-40 conda env 名**: 各机器 conda env 实际名可能不同 (如 `verl` / `verl-env` / `pyverl`), 用户配置时按实际
- **D-39 timeout 12s**: 远程 import 库 (尤其 torch_npu) 第一次冷启可能慢, 12s 不够; 07-02 1-step 单独 30s
