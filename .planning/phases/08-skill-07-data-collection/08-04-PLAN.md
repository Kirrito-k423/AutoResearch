# Phase 08 Plan 04: datalake/prometheus/push_gateway.py + manifest 写入 (D-47)

D-47 锁: manifest 含 1-step 元数据. 本 plan:
1. prometheus push (远程 pushgateway / Phase 6 REACH 探活目标)
2. RunManifest Pydantic 模型
3. writer 落 ~/.autoresearch/runs/<id>/manifest.json
4. 端到端 CLI 串 1-step + wandb sync + log collect + prom push + manifest write

## Tasks

1. **`datalake/prometheus/push_gateway.py`**
   - `push_metrics(server: ServerSpec, run_id: str, npu_count: int, pushgateway_url: str = "http://localhost:9091") -> bool`
   - 远程 `python -c "from prometheus_client import CollectorRegistry, Gauge, push_to_gateway; r = CollectorRegistry(); g = Gauge('npu_count', 'NPU count', registry=r); g.set(<npu_count>); push_to_gateway('<url>', job='ar_<run_id>', registry=r)"` 走 conda env
   - 失败 → raise PushError
   - 简化: 不引 prometheus_client SDK 硬依赖, 用 curl 推 textfile 格式 (pushgateway 接受 text/plain)
2. **`datalake/manifest/schema.py`**
   - `RunManifest` Pydantic v2: 8 字段 (run_id / started_at / finished_at / server / conda_env / lib / workdir_remote / workdir_local / one_step dict / exit_code / error / wandb_run_id / log_files / prom_metrics_file)
3. **`datalake/manifest/writer.py`**
   - `write(m: RunManifest, root: Path = Path("~/.autoresearch/runs")) -> Path` 落 `<root>/<id>/manifest.json`
   - 自动创建目录
4. **`autoresearch/collect/manifest.py`**
   - `build_manifest(run_id, spec, lib, conda_env, workdir, minimal_result, wandb_path, log_path, prom_pushed) -> RunManifest`
5. **`autoresearch/collect/cli.py`**
   - `run --server X --lib verl [--workdir /root] [--config ...]` 端到端:
     1. 生成 run_id (ULID)
     2. 调 collect_minimal
     3. 调 sync_run
     4. 调 collect_log
     5. 调 push_metrics
     6. 调 build_manifest + write
     7. 打印 manifest 路径 + 退出码
6. **CLI 注册** (`autoresearch/cli.py`): `@main.group(name="collect")` + `@collect.command(name="run")`
7. **测试**:
   - `tests/test_datalake_prometheus_push.py`: mock SSH 测 3 路径 (OK / pushgateway 不在 / 拒推)
   - `tests/test_datalake_manifest.py`: schema 验证 + writer 落盘
   - `tests/test_collect_manifest.py`: build_manifest 字段齐全
   - `tests/test_collect_cli.py`: CliRunner 测端到端 (mock 4 个下游)
8. **UAT** (`08-04-SUMMARY.md`):
   - 1 台真机 (A2-AK-225) 跑 `autoresearch collect run --server A2-AK-225 --lib verl`
   - 验证:
     - 1-step 跑通 (sum / npu_count)
     - wandb sync 成功
     - log.txt 拉到
     - prom push (本地没 pushgateway, 失败但 manifest 含 prom_pushed=False 不阻塞)
     - manifest.json 落 ~/.autoresearch/runs/<id>/manifest.json, 8 字段齐全
   - 4 台 `--all` 跑 (A2-AK-225 OK, 其他 3 台 fail 路径走通)

## 风险

- **R1 prometheus pushgateway 不在本机**: Phase 1 没装 pushgateway 容器 (Phase 6 才加). 失败路径走通即可, 不阻塞
- **R2 ULID 生成**: 用 `ulid-py` 库, 加到 pyproject deps
- **R3 端到端 CLI timeout**: 1-step 30s + wandb sync 30s + log fetch 10s + prom push 10s = 80s 上限
- **R4 config.yaml workdir 字段**: 4 台机器配 /root, 验证 config validate
