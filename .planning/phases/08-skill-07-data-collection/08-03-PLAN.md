# Phase 08 Plan 03: datalake/logs/collector.py 实时拉 (D-46 + D-47)

D-46 锁: ServerSpec.workdir 字段. D-47 锁: log 走增量拉. 本 plan:
1. schema 扩 workdir 字段
2. log collector 实现 (一次性 SFTP 拉, 非实时流)

## Tasks

1. **schema 扩字段** (`workspace-core/config/schema.py`)
   - `ServerSpec.workdir: str = Field(default="/root", description="...")`
2. **`datalake/logs/collector.py`**
   - `collect_log(run_id: str, server: ServerSpec, workdir_override: str | None = None) -> Path`
   - 步骤:
     1. 拼 remote_path = `<workdir>/runs/<run_id>.log`
     2. SFTP 拉 `sftp.open(remote_path).prefetch()` 写到 `~/.autoresearch/runs/<run_id>/log.txt`
     3. 失败 → raise LogFetchError (含原因: not found / permission / ssh)
   - 不流式: 一次性拉 (1-step 22s, 文件小)
3. **`autoresearch/collect/minimal.py` 接入** (08-01 写的, 这里扩):
   - 跑前: 远程 `mkdir -p <workdir>/runs && tee <workdir>/runs/<run_id>.log` (1-step stdio 重定向)
   - 跑后: 调 `collect_log` 拉
4. **config.yaml 更新**: 4 台 machine 配 workdir
5. **测试**:
   - `tests/test_datalake_logs_collector.py`: mock SFTP 测 3 路径 (OK / not found / permission)
   - `tests/test_config_validate.py`: 加 workdir 字段验证
6. **UAT** (`08-03-SUMMARY.md`):
   - 1 台真机 (A2-AK-225) 跑 1-step, 验证:
     - 远程 `/root/runs/<run_id>.log` 生成 + 含 SUM + NPU_COUNT
     - 本地 `~/.autoresearch/runs/<run_id>/log.txt` 跟远程一致

## 风险

- **R1 workdir 字段改 schema 影响 Phase 2-7 代码**: 所有 `ssh_exec_capture` 拼 `cd <workdir> && <cmd>` 需要 spec.workdir, 测试需全跑
- **R2 远程 /root/runs 目录可能没权限 (root 通常有)**: 验证 mkdir 失败给可读错误
- **R3 1-step stderr 包含 torch_npu 启动 log, 可能 > 100KB**: 文件大小限制? 当前不限制
