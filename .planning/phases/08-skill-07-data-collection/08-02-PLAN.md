# Phase 08 Plan 02: datalake/wandb/sync.py 离线→本地 (D-45)

D-45 锁: 远程 wandb 离线模式 + 本地 sync. 本 plan 实现 `datalake/wandb/sync.py` 把离线 run 拉回本地 wandb 服务.

## Tasks

1. **`datalake/wandb/sync.py`**
   - `sync_run(run_id: str, server: ServerSpec, workdir: str, local_runs_root: Path = Path("~/.autoresearch/runs")) -> Path`
   - 步骤:
     1. 远程 `ls <workdir>/wandb/run-<run_id>-*.wandb` 验存在 (失败 -> raise NotFoundError)
     2. SCP 整个 `<workdir>/wandb/run-<run_id>-*.wandb` 到 `local_runs_root/<run_id>/wandb/`
     3. 本地 `wandb sync <local_path>` (subprocess.run, 30s timeout) → 导入本地 wandb 服务
     4. 返回 `local_runs_root/<run_id>/wandb/`
   - 错误诊断: 4 类 (WandbNotInstalled / NoLocalServer / NoRemoteRun / SyncFailed)
2. **`datalake/wandb/__init__.py`**: 导出 `sync_run`
3. **测试**:
   - `tests/test_datalake_wandb_sync.py`: mock SSH + mock subprocess 测 4 错误路径 + happy path
4. **UAT** (`08-02-SUMMARY.md`):
   - 1 台真机 (A2-AK-225) 跑 1-step + sync, 验证:
     - 远程 wandb/<run-id> 目录生成
     - SCP 拉回本地
     - `wandb sync` 成功
     - 本地 `http://localhost:8080/<run-id>` 可见 (可选, 写 manifest 含 url)

## 风险

- **R1 wandb CLI 装在 Mac 但本地 wandb 服务没起**: Phase 1 起来, 8.1 跑下来. 失败 → 给提示 `autoresearch services start`
- **R2 SCP 路径含 wildcards**: `wandb/run-<run_id>-*.wandb` 是 glob, paramiko 没原生 glob, 用 `ls` 拿具体文件名再传
- **R3 离线 wandb 写不依赖外部服务**: 但 wandb Python 包必须在 conda_env 内, 验 import 失败
