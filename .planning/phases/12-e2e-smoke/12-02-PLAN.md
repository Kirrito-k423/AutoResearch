# Phase 12 Plan 02: Report completeness check

<objective>
Add a report completeness oracle that fails E2E when any of the required
log, wandb, or Prometheus report views are missing.
</objective>

<tasks>

<task id="12-02-T1" type="execute">
<read_first>
- `autoresearch/report/loader.py`
- `autoresearch/report/models.py`
- `tests/test_report_loader.py`
</read_first>
<action>
Create `autoresearch/e2e/report_check.py` with `check_report_completeness(...)`.
It must call `load_report_bundle(run_id, root=runs_root, prometheus_base_url=...)`
and return `(exit_code, payload)`. Payload must include checks for `html`, `log`,
`wandb`, and `prometheus`, plus `warnings`.
</action>
<acceptance_criteria>
- `check_report_completeness` returns exit code `0` only when report HTML exists and all three views are available.
- Missing log/wandb/prom view returns exit code `1` and names the missing view.
- File-not-found or loader exceptions return exit code `1` with a useful `error`.
</acceptance_criteria>
</task>

<task id="12-02-T2" type="execute">
<read_first>

<action>
Wire `check_report_completeness` into `run_e2e_smoke` after the smoke step.

checks.
</action>
<acceptance_criteria>
- E2E smoke happy path includes a `report` step with all checks passing.
- Report completeness failure stops before Archon observability.
</acceptance_criteria>
</task>
