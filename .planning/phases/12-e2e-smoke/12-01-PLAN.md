# Phase 12 Plan 01: E2E smoke CLI

<objective>
Add `autoresearch e2e smoke` to orchestrate the full local M1 path with clear
step diagnostics and duration/Archon observability gates.
</objective>

<tasks>

<task id="12-01-T1" type="execute">
<read_first>
- `autoresearch/orchestrator/checks.py`
- `autoresearch/orchestrator/smoke.py`
- `autoresearch/orchestrator/models.py`
- `autoresearch/cli.py`
</read_first>
<action>
Create `autoresearch/e2e/smoke.py` with `run_e2e_smoke(...)`. It must run
`run_check_all`, then `run_smoke`, then Archon observability and duration gates.
Use `StepResult` helpers for normalized step payloads. Return `(exit_code, payload)`
with `ok`, `command`, `server`, `lib`, `run_id`, `report`, `failed_step`,
`elapsed_seconds`, `summary`, and `steps`.
</action>
<acceptance_criteria>
- `autoresearch/e2e/smoke.py` contains `def run_e2e_smoke(`.
- Readiness failure sets `failed_step` to `readiness` and skips downstream steps.
- Smoke failure sets `failed_step` to `smoke`.
- Duration failure sets `failed_step` to `duration`.
- Progress stages include `e2e.smoke.start`, per-step start/result, and `e2e.smoke.result`.
</acceptance_criteria>
</task>

<task id="12-01-T2" type="execute">
<read_first>
- `autoresearch/cli.py`
- `autoresearch/collect/cli.py`
- `autoresearch/report/cli.py`
</read_first>
<action>
Add `autoresearch e2e smoke` CLI. Options: `--server` required, `--lib`
(`verl|veomni`) default `verl`, `--config`, `--workdir`, `--timeout`,
`--run-id`, `--pushgateway-url`, `--prometheus-url`, `--prometheus-wait`,
`--archon-url`, `--max-duration`, and `--open`. The command must print one
JSON object and exit with the orchestrator exit code.
</action>
<acceptance_criteria>
- `uv run autoresearch e2e smoke --help` lists `--server`, `--max-duration`, and `--archon-url`.
- CLI wrapper catches exceptions and returns JSON with `failed_step=orchestrator`.
</acceptance_criteria>
</task>

<task id="12-01-T3" type="execute">

- `tests/test_orchestrator_smoke.py`
- `tests/test_cli.py`

<acceptance_criteria>
