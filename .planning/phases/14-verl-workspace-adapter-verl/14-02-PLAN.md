# Phase 14 Plan 02: Verl Docker, Data Prep, Provenance, And Remote Formal Runner

## Objective

Build the Verl-specific adapter that can prepare multimodal geometry3k data, construct Ascend Docker commands, capture provenance, and execute the strict sync/async length matrix on a selected remote host.

## Threat Model

- **T-14-02-01 shell injection:** Docker and remote commands must quote every path/value and avoid user text in shell positions.
- **T-14-02-02 silent data downgrade:** Dataset prep must fail if image fields cannot be preserved.
- **T-14-02-03 false success:** Runner must mark top-level `ok=false` if any matrix row fails.
