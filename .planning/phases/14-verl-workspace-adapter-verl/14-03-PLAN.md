# Phase 14 Plan 03: `autoresearch run verl-case` Orchestration And Local Artifacts

## Objective

Expose the formal case as `autoresearch run verl-case`, reuse readiness skills for machine selection, execute the Verl adapter, and persist every formal-run artifact locally with a single JSON result.

## Threat Model

- **T-14-03-01 CLI protocol break:** stdout must remain exactly one JSON object; progress belongs on stderr.
- **T-14-03-02 hard-coded host:** default server selection must come from config/readiness, with explicit `--server` override allowed.
- **T-14-03-03 artifact loss:** ok=true must require local config/provenance/matrix/log/manifest/report paths to exist.

The function should emit progress stages `orch.verl_case.start`, `orch.verl_case.readiness`, `orch.verl_case.prepare`, `orch.verl_case.run`, `orch.verl_case.collect`, `orch.verl_case.report`, and `orch.verl_case.result`.   <verify>
    uv run pytest -q tests/test_orchestrator_verl_case.py -k orchestration
  </verify>
  <acceptance_criteria>

    - With one failed matrix row, payload contains `ok == false` and `failed_step == "matrix"`.
  </acceptance_criteria>
