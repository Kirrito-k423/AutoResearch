# Phase 14 Plan 01: Formal Case Config, Matrix, Evaluation, And Provenance Models

## Objective

Create the deterministic local contract for `verl-case`: typed config defaults, strict length matrix, immutable config snapshots, answer evaluation helpers, and manifest/provenance schema extensions.

## Threat Model

- **T-14-01-01 secret leakage:** Immutable config snapshots and provenance must not include HF/GitHub tokens, SSH passwords, or BMC passwords.
- **T-14-01-02 false completion:** Matrix generation must not silently omit 16k or async rows.
- **T-14-01-03 unverifiable accuracy:** Answer extraction must be deterministic and tested against geometry-style answers.
