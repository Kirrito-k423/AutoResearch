# Phase 03 / Customer-Config Skill — Context (6 REQ + 8 decisions)

## Scope

Skill 01 of the 8-skill loop. 用户 1 步完成 "生成/校验/查看" config, 敏感字段不裸奔.

**Goal**: 用户能用一个命令生成、校验、查看配置；敏感字段不裸奔。
**Depends on**: Phase 2 (workspace-core/config + workspace-core/secrets)
**Requirements**: CFG-INIT-01..03, CFG-VAL-01..02, CFG-SHOW-01

## REQ 拆分

| REQ | 含义 | Plan |
|---|---|---|
| CFG-INIT-01 | `ar-config init` 生成 `config/config.yaml` 模板 | 03-01 |
| CFG-INIT-02 | 模板含中文注释, 每个字段都有说明 | 03-01 |
| CFG-INIT-03 | 已存在 config 时拒绝覆盖 (除非 `--force`) | 03-01 |
| CFG-VAL-01 | `ar-config validate` 校验现有配置 | 03-01 |
| CFG-VAL-02 | 校验失败指出哪个字段、什么问题、怎么修 | 03-01 |
| CFG-SHOW-01 | `ar-config show` 打印配置 (敏感字段脱敏 `***`) | 03-02 |

## Decisions (locked)

### D-01: CLI 命名 `autoresearch config` (而非 `ar-config`)

- ROADMAP 写 `ar-config` 是 spec 缩写, 实际命令是 `autoresearch config {init, validate, show}`
- 跟 `autoresearch services {status, start, stop}` + `autoresearch ping` 树状对齐
- AGENTS.md 8 步循环表已经定名 `autoresearch config`

### D-02: 模板源 = `config/config.example.yaml` (已存在, 02-04 创建)

- 不重新手写模板, 复制 example 即可 (DRY: 维护点只 1 个)
- `init` 复制后:
  - 顶部加 `# Generated by autoresearch config init at <ISO time>` 标记
  - servers 段保留注释示例, 但**清掉** 实际的 host/user (避免误用)
  - 加 `# 编辑完跑: autoresearch config validate` 引导

### D-03: 已存在 config 时 exit 3, `--force` 覆盖

- exit code 3 = "config exists, use --force" (区别于 0=ok / 1=fail / 2=missing)
- `--force` 强制覆盖, 二次确认写 "⚠ 覆盖现有 config/config.yaml (--force)"
- 不弹交互式 y/n (CI/脚本友好)

### D-04: validate 走 `workspace_core.config.from_path()` (Phase 2 沉淀)

- 复用 02-02 的 `from_path` + `_format_validation_error` 中文错误
- 校验成功: 打印 "✅ config/config.yaml 校验通过" + 服务数 / 网络探针数等简短摘要
- 校验失败: 抛 ConfigError, exit 1 (跟其它子命令一致)

### D-05: 敏感字段脱敏策略

- 字段名匹配正则 `(?i)(password|secret|token|key|credential)` → 值显示 `***`
- 例外: `identity_file` 不算敏感 (只是路径, 不是 key 内容)
- 例外: `keyring` 字段值也是 `***` (走 keyring 名字, 不是值)
- 嵌套 dict / list 递归脱敏
- 保留**字段名**让用户能定位

### D-06: init 默认从 `./config/config.yaml` 写到 cwd, `--config` flag 可覆盖

- 跟 from_path 优先级一致: path 参数 > env `AUTORESEARCH_CONFIG` > 默认
- init 时如果 `--config` 指定但父目录不存在 → 自动 `mkdir -p`

### D-07: 子命令 stdout 仍走 D-04e: 唯一 JSON (机器读) / 默认人类可读 (中文)

- `autoresearch config init [--force] [--config PATH]`: 人类可读 (中文提示 + 文件路径)
- `autoresearch config validate [--config PATH] [--json]`: 默认人类可读; `--json` 返 JSON
- `autoresearch config show [--config PATH] [--json]`: 默认人类可读; `--json` 返 JSON
- 进度协议 `__AR_PROGRESS__=` 走 stderr (D-04d)

### D-08: 中文注释从 `config/config.example.yaml` 沿用, init 时同步过去

- example.yaml 已有 5 段 (servers/network/log/wandb/version) + 中文引导
- init 复制 example 时保留所有注释 (YAML `#` 注释会被 PyYAML 丢, 所以走"文件级别复制"而不是"yaml 解析后重写")
- 用 `shutil.copyfile` 简单粗暴地复制 example, 然后插入 generated-by header

## Reused (Phase 2 沉淀)

- `workspace_core.config.from_path` / `from_yaml` / `ConfigError` (CFG-VAL-01..02 直接复用)
- `workspace_core.config.Config` / `ServerSpec` 等 schema (init 模板跟它对齐)
- `workspace_core.secrets.resolve_secret` (init 后用户跑 `validate` 才发现密码占位符没解, 不算 init 的范围)

## Non-Goals (M1 范围外)

- 远程 git 同步 config (留给 v1.1)
- 加密 config 落盘 (走 gitignore + keyring 就够)
- GUI 编辑器
- 模板多版本 (v1/v2 schema migration)

## Risks / Landmines

- **R-01**: example.yaml 改了字段但 init 没同步 → 解决: DI-01 注释里说"edit example.yaml → re-init"
- **R-02**: 用户跑 init 后改了密码明文 → 提醒加 `<env:VAR>` 占位符
- **R-03**: validate 跑在 dummy 数据上返一堆错 → 提供 `--quiet` 减少 noise (M1 可选)

## Verification (M1 完成标准)

1. `autoresearch config init` 跑通, 生成 config/config.yaml 含中文注释
2. `autoresearch config init` 第二次返 exit 3
3. `autoresearch config init --force` 覆盖
4. `autoresearch config validate` 成功 (用 example.yaml 内容) + 打印摘要
5. `autoresearch config validate` 失败 (故意改坏) + 中文错误含字段名
6. `autoresearch config show` 打印, 敏感字段显示 `***`
7. `autoresearch config show --json` 返 JSON
8. 6+ pytest PASS
