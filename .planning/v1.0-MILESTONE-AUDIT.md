## User Quick Start Assessment

Current README quick start gets the user only to service startup and health:

```bash
archon serve --port 8088 &
autoresearch services start
autoresearch services status
```

That is useful, but it is not yet the most important quick start. The best validated user case should be documented as:

```bash
uv run autoresearch services start
archon serve --port 8088
uv run autoresearch services status --json
uv run autoresearch check all --server A2-AK-225 --stack-lib verl
uv run autoresearch run smoke \
  --server A2-AK-225 \
  --lib verl \
  --timeout 60 \
  --pushgateway-url http://127.0.0.1:17891 \
  --open
```

Or, for the strongest single command:

```bash
uv run autoresearch e2e smoke \
  --server A2-AK-225 \
  --lib verl \
  --timeout 60 \
  --pushgateway-url http://127.0.0.1:17891 \
  --open
```

Validated exemplar:

- Run id: `01KV62JVH0N3ZRVRMH4PYWF1VB`
- Report: `/Users/Zhuanz/.autoresearch/runs/01KV62JVH0N3ZRVRMH4PYWF1VB/report.html`
- E2E time: `146.673s`
- Report completeness: html + log + W&B + Prometheus all present
